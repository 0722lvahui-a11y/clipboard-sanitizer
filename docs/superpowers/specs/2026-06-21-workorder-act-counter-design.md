# 工单 act_ 账户计数工具 — 设计文档

## 概述

一个 macOS 桌面程序（.app 双击打开），自动从 Excel/CSV 文件中读取工单编号，到 medialhub 网站逐个查询每个工单详情中的 act_ 开头账户数量，汇总结果并支持一键复制。

## 用户故事

1. 选择查询时间段（开始日期 ~ 结束日期）
2. 选择本地的 Excel (.xls/.xlsx) 或 CSV 文件
3. 可选：筛选特定问题类型（默认"绑定/解绑BM报错重试"）
4. 点击「开始」，程序在后台隐形浏览器中自动：
   - 打开 medialhub 工单列表页
   - 检查是否已登录，未登录则跳出提醒
   - 对每个工单编号：填入日期、填入编号、点击查询、点击详情、统计 act_ 账户数
5. 实时显示进度，最终展示结果表格 + 一键复制按钮

## 技术方案

### 界面层 — tkinter
- 与现有「黑毛猪净化器」技术栈一致
- 窗口标题：「工单act_计数工具」
- 日期选择：两个输入框（开始/结束，格式 YYYY/MM/DD）
- 文件选择按钮 → 系统文件对话框
- 筛选项输入框（预填"绑定/解绑BM报错重试"）
- 「开始查询」按钮 → 触发后台任务
- 进度条 + 状态文字
- 结果区域：可滚动的表格（工单编号 | act_账户数 | 状态）
- 「一键复制」按钮 → 复制为 TSV 格式到剪贴板

### 自动化层 — Playwright
- 使用 `playwright` 的 Chromium headless 模式（后台隐形浏览器）
- 工作流程：
  1. 打开 `https://mediahub.meetsocial.cn/account/workorder`
  2. 检测页面是否已登录（检查是否存在登录表单或跳转到登录页）
  3. 若未登录 → 通知用户 → 暂停
  4. 对于每个工单编号：
     a. 填入开始日期、结束日期
     b. 填入工单编号到搜索框
     c. 点击查询按钮
     d. 等待结果加载
     e. 点击工单进入详情页
     f. 在「基本信息」区域查找所有 act_ 开头的文本
     g. 计数
     h. 返回工单列表页
  5. 汇总结果

### 文件读取
- CSV：Python 内置 csv 模块
- Excel：openpyxl（.xlsx）+ xlrd（.xls）
- 读取第一列作为工单编号，可选按第二列过滤

### 打包
- PyInstaller --onefile --windowed
- macOS .app bundle（结构参考 ClipboardSanitizer）
- GitHub Actions 自动构建

## 数据流

```
CSV/Excel 文件
    ↓ 读取 + 过滤
工单编号列表 (list[str])
    ↓ 逐个处理 (Playwright)
每个工单 → [查询 → 详情 → 数 act_]
    ↓ 实时更新
结果列表 [ {工单编号, act_数量, 状态} ]
    ↓
UI 表格 + 一键复制
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 未登录 | 弹窗提示"请先在浏览器中登录mediahub"，程序暂停 |
| 网络超时 | 单个工单最多重试2次，失败标记为"超时" |
| 页面加载超时 | 等待30秒，超时跳过该工单，标记"加载超时" |
| 工单编号不存在 | 标记 act_数量为 0 或"未找到" |
| 文件格式不支持 | 弹窗提示，不崩溃 |
| 文件为空/无匹配行 | 弹窗提示"未找到符合条件的工单" |
| 用户中途关闭 | 优雅退出，清理浏览器进程 |

## 安全原则

- **不记录、不存储任何密码和登录信息**
- 检测到未登录状态时，只弹窗提醒用户手动登录
- 所有操作在本地完成，不上传任何数据
